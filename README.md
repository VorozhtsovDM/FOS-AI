## 1. О дисциплине

#### Цель дисциплины
Цель дисциплины - изучение студентами современных методов машинного обучения, инструментов и технологий анализа данных; формирование профессиональных компетенций студентов по КРМ ИТМО в области компетенций по машинному обучению для ролей: Data Analyst и ML Engineer.

#### Основные задачи дисциплины
-	изучение теоретических основ машинного обучения; знакомство с различными научными направлениями в этой сфере;
-	изучение современных технологий сбора данных для машинного обучения;  
-	изучение методов и технологий предварительного анализа и визуализации данных.
-	изучение методов машинного обучения с учителем: регрессия, классификация;
-	изучение методов машинного обучения без учителя: кластерный анализ;
-	формирование навыков оценки и повышения качества моделей машинного обучения;
-	формирование навыков сравнения и выбора моделей машинного обучения для практических задач;
-	формирование навыков проектирования воспроизводимых пайплайнов обработки данных и обучения моделей;
-	систематизация и закрепление практических навыков и умений по решению интеллектуальных задач с использованием инструментов машинного обучения на языке Python.

#### Структура и содержание
Дисциплина состоит из 2 модулей: 1. Основы машинного обучения; 2. Классическое машинное обучение. Обучение включает лекции, лабораторные работы, проект, самостоятельную работу и завершается зачетом.

#### Пререквизиты и связи с другими дисциплинами
Изучение дисциплины базируется на знаниях, полученных обучающимися при освоении следующих дисциплин:\
-	Введение в искусственный интеллект
-	Теория вероятностей
-	Алгоритмизация и программирование
-	Объектно-ориентированное программирование
-	Программирование на языке Python

Для освоения дисциплины «Машинное обучение» студент должен\
Знать:
-	основные понятия искусственного интеллекта;
- основы объектно-ориентированного программирования;
-	структуры данных и алгоритмы их обработки;
-	методы теории вероятностей и математической статистики.

Уметь:
-	делать анализ поставленной задачи,
-	выбирать необходимые структуры данных и объектов,
-	составлять алгоритм решения задачи,
-	разбивать задачи на подзадачи;
-	разрабатывать программные средства реализации методов анализа данных на языке Python.

Владеть:
-	навыками работы в инструментальных средах программирования;
-	навыками проектирования пользовательских интерфейсов.

#### Планируемые результаты обучения

По завершении дисциплины обучающийся сможет: 
- применять модели машинного обучения, соответствующие поставленной задаче;
- проводить оценку качества моделей машинного обучения и использовать инструменты, позволяющие повысить качество моделей машинного обучения;
- проводить сравнительный анализ моделей машинного обучения и обосновывать выбор наиболее подходящей модели;
- применять подходы к решению задач искусственного интеллекта с использованием алгоритмов машинного обучения;
- использовать методы и приемы анализа и структурирования сложных программ;
- создавать воспроизводимые пайплайны обработки данных и обучения моделей.

Полное описание приведено в [РПД](docs/rpd.md).

## 4. Итоговая оценка

Для всех оценочных средств применяется единая 10-балльная шкала. Ключевые опорные точки шкалы соответствуют уровням освоения роли по КРМ:
Балл|Уровень|Характеристика (по КРМ)|5-балльная шкала
|---|---|---|---|
10|Аналитик-визионер / Инженер-архитектор|Формирует стратегию, стандартизирует практики роли, признаки научной новизны или высокой практической ценности|Отлично
9|Аналитик-руководитель проекта / Ведущий инженер|Формирует техническое задание на новые аналитические проекты, обосновывает выбор методов и инструментов анализа данных|Отлично
8|Разработчик аналитического проекта / Инженер-разработчик|Самостоятельно создает новые аналитические проекты, обосновывает выбор методов и инструментов анализа данных|Отлично
7|Уверенный аналитик / Уверенный инженер|Самостоятельно реализует роль аналитика данных на прикладной задаче, применяет лучшие практики|Хорошо
6|Самостоятельный аналитик / Самостоятельный инженер|Самостоятельно реализует аналитический проект по типовому паттерну|Хорошо
5|Начинающий аналитик / Начинающий инженер|Реализует аналитический проект по инструкции, требует консультаций по нетиповым решениям|Удовлетворительно
4|Исполнитель|Применяет инструменты по инструкции, минимально приемлемый уровень для роли|Удовлетворительно
3|Ниже порога|Частично освоил инструменты, серьёзные пробелы|Неудовлетворительно
2|Фрагментарно|Демонстрирует лишь отдельные элементы роли|Неудовлетворительно
0–1|Не сформирован|Компетенция не подтверждена оценочным средством|Неудовлетворительно

#### Перечень оценочных средств
Код|Оценочное средство|Форма проведения|Тип оценивания
|---|---|---|---|
ОС-1|[Лабораторные работы](docs/laboratory.md)|Практическое задание с защитой|формирующее
ОС-2|[Кейс-задание (case study)](docs/case-study.md)|Разбор прикладного кейса, письменный отчёт|формирующее
ОС-3|[Peer-review заданий лабораторных работ](docs/peer-review.md)|Взаимное рецензирование по чек-листу|формирующее
ОС-4|[Защита итогового проекта](Project/README.md)|Презентация и демонстрация решения|суммирующее
ОС-5|[Тест (промежуточная аттестация)](exam/README.md)|Тест с открытыми и закрытыми вопросами|формирующее

Итоговая оценка по дисциплине формируется как средневзвешенное баллов по оценочным средствам:
-	лабораторные работы — 25%;
-	кейс-задание — 15%;
-	peer-review (участие + качество рецензий) — 10%;
-	тест промежуточной аттестации — 25%;
-	защита итоговго проекта перед комиссией — 25%;
Итоговый проект выполняется по [кейсам индустриальных партнёров](resources/tasks/README.md#кейсы-индустриальных-партнёров); описание проектной работы — в разделе [Проектная работа](Project/README.md).

Дисциплина считается освоенной при итоговой оценке не ниже 4,0 баллов по десятибальной шкале.
