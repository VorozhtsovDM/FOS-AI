Процедура оценивания состоит из четырех основных этапов
#### 1. Распределение работ
Слепое рецензирование: Работы распределяются анонимно (студенты не знают, чью работу они проверяют).\
Кросс-проверка: Каждый студент проверяет от 2 до 3 работ своих однокурсников.
#### 2. Критерии оценки (Рубрикатор)
Для объективности рецензентам выдается оценочный лист по криткриям. По каждому критерию рецензент выставляет балл от 0 до 10 и оставляет содержательный комментарий (2–5 предложений). Комментарий обязателен для баллов ≤ 5 и ≥ 9.
| Код	| Критерий	| Пояснение	| Балл (0–10) |
|---|---|---|---|
| P1 | Ясность постановки задачи | Задача понятна без знакомства с автором; целевой пользователь и сценарии зафиксированы. | _____ |
| P2 | Обоснованность выбора ИИ-подхода | Выбор класса моделей и инструментов обоснован относительно задачи и данных. | _____ |
| P3 | Работа с данными | Источники, качество, предобработка данных описаны и проверяемы. | _____ |
| P4 | Архитектура и её описание | Схема архитектуры соответствует коду, потоки данных ясны, ИИ-компонент локализован. | _____ |
| P5 | Работоспособность MVP | MVP запускается по инструкции; ошибки обрабатываются; демо-сценарии проходят. | _____ |
| P6 | Эксперименты и метрики | Метрики релевантны; результаты воспроизводимы; выводы аргументированы. | _____ |
| P7 | Репозиторная документация | README, PROJECT_BRIEF, docs/architecture.md; понятны стороннему инженеру. | _____ |
| P8 | Ограничения, риски, этика | Границы применимости, риски и этические аспекты явно проработаны. | _____ |
| P9 | Соответствие индикаторам роли КРМ | Проект отвечает индикаторам выбранной студентом ИИ-роли. | _____ |
| P10 | Общая инженерная культура | Структура, читаемость кода, тесты, версии зависимостей, лицензия. | _____ |
#### 3. Процесс ревью и заполнение формы
Ревьюер изучает работу и выставляет баллы по каждому пункту, а также оставляет конструктивные комментарии:
- Указывает на конкретные ошибки, а не просто констатирует их наличие.
- Предлагает варианты исправления.Отмечает сильные стороны работы.
#### 4. Апелляция и выставление итоговой оценки
Если автор не согласен с оценкой ревьюера, он может подать апелляцию преподавателю. Окончательную оценку (итоговый балл) за лабораторную работу выставляет преподаватель на основе выставленных рецензий и собственного выборочного контроля.
